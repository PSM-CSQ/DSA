## 动态规划

**动态规划问题的一般形式就是求最值**。比如说求最长递归子序列，最小编辑距离等等。

求解动态规划问题的**核心就是穷举**。把所有可行的答案穷举出来，然后在其中在最值。

首先，动态规划在穷举的过程中，存在“**重叠子问题**”的现象。如果暴力穷举的话会导致效率低下，所以我们可以通过“备忘录”或者“DP Table”来优化穷举过程，避免重复的计算。

而且，动态规划问题一定会具备“**最优子结构**”，才能通过子问题的最值得到原问题的最值。

虽然动态规划的核心思想就是穷举求最值，但是往往穷举所有可行解并不简单，只有找出正确的“**状态转移方程**”，才能正确穷举。

重叠子问题、最优子结构、状态转移方程就是动态规划三要素。在实际解决问题的过程中，写出状态转移方程是最难的。

思维框架，辅助思考状态转移方程：

**明确 base case -> 明确「状态」-> 明确「选择」 -> 定义 dp 数组/函数的含义**。

```
# 初始化 base case
dp[0][0][...] = base
# 进行状态转移
for 状态1 in 状态1的所有取值：
    for 状态2 in 状态2的所有取值：
        for ...
            dp[状态1][状态2][...] = 求最值(选择1，选择2...)
```
